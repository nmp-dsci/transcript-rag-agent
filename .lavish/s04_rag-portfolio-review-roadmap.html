Where this sits against 2026 state of the art

The center of gravity has moved from "which retrieval trick" to systems + measurement. Hybrid search, reranking and a RAGAS harness are now assumed — with PR #5 merged, transcript·lab sits exactly at the 2026 baseline. Differentiation now comes from measured adaptive retrieval, the full eval doctrine, production operations, and one hard technique done deeply.

2026 differentiators — the roadmap targets

  • NO Adaptive routing: simple→cheap path, complex→agentic, with measured cost-quality per route → P3
  • NO Full eval doctrine: recall@k/MRR/NDCG, synthetic testsets, judge-vs-human validation, CI gating → P0+P1
  • NO Production ops: OTel GenAI tracing, real cost/answer, semantic cache, incremental indexing → P2
  • NO Indirect prompt-injection red-team over retrieved content — the rarest RAG portfolio item → P3
  • NO GraphRAG / temporal knowledge graph — best technique-to-domain fit for a channel corpus → P4
  • NO Late chunking — transcripts are its ideal case (pronouns destroy naive chunk embeddings) → P1
  • NO Multimodal retrieval over keyframes/slides → P5
  • HALF Long-context vs RAG: raw baseline exists for one video, no corpus-scale bake-off → P5
Note: the chunk graph is a visualization, not GraphRAG

The kNN cosine-similarity graph (src/rag/graph.py) is a genuinely nice corpus-inspection tool — exact dense similarity, deterministic PCA layout, query-neighbourhood highlighting. But it never participates in retrieval. Microsoft-style GraphRAG builds an entity/claim graph with community summaries and answers global questions vector search can't ("what themes run across this channel", "how did this claim evolve"). That upgrade is Project 4 — and it happens to be the original project goal: trend tracking across related videos.

Adaptive & Secure Agentic RAG

~2 weeks

You already measured that the agent costs 7× more per quality point than single-hop (0.033 vs 0.234 composite/1k tokens). Build the router that exploits that finding — then attack your own system, because a transcript corpus is untrusted input and nobody else's portfolio shows injection defense.

Build

  • Query router: classify simple vs multi-hop questions, route to single-hop vs recursive/agentic; report cost-quality per route on the golden set — the "adaptive RAG" pattern
  • Retrieval grader + CRAG-style corrective fallback: grade retrieved chunks, rewrite the query or widen scope when they're weak
  • Conversational memory for rag_agent (parity with rag_llm)
  • Injection red-team: poisoned-transcript fixtures ("ignore your instructions…" inside a video), promptfoo OWASP LLM01 suite, then defenses — content quarantining, instruction hierarchy, output filtering — with before/after attack success rates

Evidence produced

  • Routing decision table: per-route cost, quality, latency
  • A red-team report with measured attack-success reduction

Market signal

  • Agentic AI: 80% of JDs
  • Security: OWASP LLM01, "the most exploited vector in production" — rarest portfolio differentiator
  • Mastercard: "balance performance/cost/security/maintainability"
